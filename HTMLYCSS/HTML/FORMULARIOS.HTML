Un formulario es un conjunto de campos (como cajas de texto, botones, listas desplegables, casillas de verificación, etc.) que permiten al usuario ingresar y enviar datos a un servidor o procesarlos en la página.

Etiquetas de Formularios

Todas las etiquetas del formulario deben estar ubicadas dentro de las etiquetas <form>.

  • Nombre
  • Correo
  • Contraseña
  • Fecha de nacimiento
  • Rango
  • Color
  • Número
  • Enviar

Ejemplo de Formulario Completo

A continuación se muestra un formulario que combina todos los tipos de input vistos: texto, contraseña, email, fecha, rango, color, número y botón enviar.

Formulario completo:

            <form>

            <!-- Nombre -->
            <label for="nombre">Nombre:</label>
            <input type="text" 
            id="nombre" 
            name="nombre" 
            size="40" 
            title="Escribe tu nombre real. Es el que aparecerá en tu perfil." 
            autofocus 
            required />

            <!-- Contraseña -->
            <label for="password">Contraseña:</label>
            <input type="password" 
            id="password" 
            name="password" 
            size="7" 
            title="Escribe la contraseña que desees..." 
            required />

            <!-- Correo Electrónico -->
            <label for="mail">Correo:</label>
            <input type="email" 
            id="mail" 
            name="mail" 
            size="40" 
            placeholder="usuario@dominio.com" 
            title="Escribe tu correo electrónico..." 
            required />

            <!-- Fecha de Nacimiento -->
            <label for="nacimiento">Fecha de nacimiento:</label>
            <input type="date" 
            id="nacimiento"
            name="nacimiento" 
            value="2000-01-01"
            title="La fecha en la que naciste, ya que..."/>

            <!-- Rango -->
            <label for="adiccion">¿Cuántos cómics de Flash Gordon lees a la semana?:</label>
            <input type="range" 
            id="adiccion"
            name="adiccion"
            min="1"
            max="7"
            value="2"
            step="1"
            title="Necesitamos saber..." />

            <!-- Color -->
            <label for="color_preferido">Color favorito de las mayas de Flash Gordon:</label>
            <input type="color" 
            id="color_preferido"
            name="color_preferido"
            value="#FF0000"
            title="Con esta votación..." />

            <!-- Número -->
            <label for="comics">¿Cuántos cómics de Flash Gordon tienes actualmente?:</label>
            <input type="number" 
            id="comics"
            name="comics"
            min="100"
            max="9000"
            value="100"
            step="1"
            title="De momento el ganador..." />

            <!-- Enviar -->
            <input type="submit" value="Enviar los datos" />

            </form>

  • Texto:Campo para ingresar texto. size define su longitud, title muestra un mensaje al pasar el cursor, name identifica el dato al enviar el formulario, autofocus selecciona el campo por defecto y required lo hace obligatorio.
  • Contraseña:El texto ingresado se oculta. También puede ser obligatorio con required.
  • Correo electronico:Valida automáticamente el formato de correo. placeholder muestra un texto inicial que desaparece al escribir. required lo hace obligatorio.
  • Fecha:Permite seleccionar una fecha mediante calendario. value define la fecha inicial por defecto.
  • Rango:Barra deslizante para elegir un valor numérico entre min y max. step define el incremento y value el valor inicial.
  • Color:Selector de color con valor inicial definido por value en hexadecimal.
  • Numero:ermite ingresar valores numéricos dentro de un rango (min, max). step define el incremento y value el valor inicial.
  • Enviar:Botón que envía los datos del formulario. El texto del botón se define con value.

Otras etiquetas o inputs existentes

Existen múltiples tipos de input type, como son:

Ejemplo en CSS de sombras:

Button: Crea un botón 'vacío'. La acción que se ejecutará cuando el usuario haga clic en él. La acción a ejecutar se tendrá que añadir posteriormente con JavaScript indicando su nombre de id. 

                <input type="button" value="Vótame" id="boton1"/>

Checkbox: Crea cuadros de selección en el que se puede seleccionar más de un valor. Como es posible marcar múltiples checkbox, cada uno tiene que tener un name diferente. 

                <input type="checkbox" name="destino1" value="Damasco" /> Damasco (Siria)
                <input type="checkbox" name="destino2" value="Kabul" checked /> Kabul (Afganistán)
                <input type="checkbox" name="destino3" value="Belgrado" /> Belgrado (Serbia)

File: Permite que el usuario pueda seleccionar un fichero de su disco duro. Además, es posible indicar un filtro para poder seleccionar únicamente un tipo de fichero determinado. (Posteriormente para subirlo al servidor o realizar cualquier otra acción con dicho fichero se tendrá que recurrir a JavaScript). 

                <input type="file" name="fichero" accept="image/*" />

Radio: Crea círculos de selección en el que únicamente se puede seleccionar uno de ellos. Todos los radio button tienen que tener el mismo valor en name. 

                <input type="radio" name="sexo" value="Mujer" checked /> Mujer
                <input type="radio" name="sexo" value="Hombre" /> Hombre y sus derivados
                <input type="radio" name="sexo" value="NS/NC" /> No estoy muy seguro/a

Tel: Introduce un número de teléfono. Aunque este type no tiene autovalidación (se puede escribir lo que se desee) ofrece importante información al navegador del tipo de información a introducir. Así, navegadores en iPhone4 (iOs 4.2) o Android (HTC Desire) cuando reciben un type="tel" ofrecen un teclado numérico especial para facilitar la introducción de este tipo de información. 

                <input type="tel" name="telefono" />

Estilos CSS y autovalidación visual

Con CSS3 podemos maquetar los inputs de los formularios para que sean más agradables a la vista y además ofrecer información visual para saber qué campos tienen valores válidos y qué valores son incorrectos o bien son obligatorios y que por lo tanto deben ser rellenados.

Ejemplo en CSS de sombras:

Maquetación CSS: Para maquetar y dar un estilo determinado al formulario podemos utilizar los selectores referentes a cada uno de los elementos que componen el formulario form, input, fieldset, legend, label... 

                form{
                background-color:orange;
                color:black;
                }
                input{
                color:grey;
                }
                fieldset{
                border-radius:5px;
                border:1px dotted green;
                }

Autovalidacion: Para que el formulario pueda resaltar visualmente qué campos obligatorios es necesario rellenar, que campos son válidos o qué campos tienen valores incorrectos utilizamos el selector :invalid, mientras que el selector [required]{ señala aquellos campos que son obligatorios y que los valores introducidos son correctos y :valid se encarga de gestionar aquellos campos que son correctos. 

                input[required]{
                border-color:blue;
                box-shadow:0px 0px 5px blue;
                }
                input:invalid{
                border-color:red;
                box-shadow:0px 0px 5px blue;
                }
                input:valid{
                border-color:green;
                box-shadow:0px 0px 5px green;
                }
