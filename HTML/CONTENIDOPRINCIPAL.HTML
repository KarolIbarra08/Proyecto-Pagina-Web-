Conceptos Importantes

Página Web

Definición

En términos técnicos, una página web es un documento digital diseñado para ser visualizado en un navegador (como Chrome, Safari o Firefox). Está compuesta principalmente por código que le indica al navegador qué mostrar (texto, imágenes, videos) y cómo debe lucir.

Importancia

La importancia del desarrollo de páginas web no radica únicamente en lo comercial, sino en que:

  • Traduce la arquitectura del sistema al usuario: Todo lo que ocurre en backend (reglas de negocio, bases de datos, servicios) solo tiene valor si puede representarse e interactuarse correctamente en el cliente.
  • Manifiesta requisitos no funcionales: Rendimiento, seguridad, escalabilidad y mantenibilidad se evidencian directamente en la interfaz.
  • Regula el tráfico hacia el sistema:
    • Frontend mal diseñado:
      1. Genera solicitudes innecesarias.
      2. Incrementa latencia.
      3. Sobrecarga el backend.
    • Frontend bien diseñado:
      1. Optimiza peticiones.
      2. Reduce consumo de recursos.
      3. Mejora eficiencia sistémica.

Es por eso que, con el paso del tiempo, se han desarrollado dos ítems importantes: Arquitectura Web y Arquitectura Frontend. Su importancia radica en que es el punto donde se concretan los procesos del sistema y se evidencian requisitos como rendimiento, seguridad y escalabilidad. Debido a esta criticidad, se requiere una Arquitectura Web que garantice comunicación y manejo adecuado de datos, así como una Arquitectura Frontend que organice el código y controle la complejidad del cliente.

Arquitecturas

Arquitectura Web

La arquitectura web es la estructura general que define cómo interactúan todos los componentes de una aplicación web: cliente, servidor, bases de datos y servicios externos.

Desde un punto de vista técnico, establece:

  • Cómo el navegador se comunica con el servidor mediante HTTP/HTTPS.
  • Cómo se procesan las solicitudes y respuestas.
  • Cómo se organiza el backend (controladores, servicios, modelos).
  • Cómo se integran APIs o microservicios.

Arquitectura Frontend

La arquitectura frontend se enfoca específicamente en la organización interna de la parte visual e interactiva que ve el usuario en el navegador.

Desde un punto de vista técnico, establece:

  • Cómo se estructuran los archivos (HTML, CSS, JavaScript).
  • Cómo se organizan los componentes.
  • Cómo se maneja el estado de la aplicación.
  • Cómo se comunican los módulos entre sí.
  • Qué patrón de diseño se utiliza (ejemplo: MVC).
